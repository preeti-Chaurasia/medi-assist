[1:00 — ARCHITECTURE]
(Cue: show assets/architecture_diagram.png)
Let's look at how the system processes information. Every single incoming user request first enters our strict Security Checkpoint node. If any malicious prompt injection or dangerous substance request is caught, the workflow routes to a dedicated Security Violation Handler, shielding the core system. Clean requests are safely stripped of sensitive PII and passed to the Central Orchestrator. From there, the Orchestrator intelligently delegates tasks to specialized sub-agents: the Symptom Analyzer or the Prescription Safety Helper, which pull real-time data from our custom MCP Server. If a high-risk medication reminder needs scheduling, the workflow triggers a Human-in-the-Loop pause node, waiting for explicit confirmation before altering state.
